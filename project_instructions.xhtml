Dawnbreakers Project Instructions

Core Rules

  • All content must match the 11 uploaded note files.
  • When referring to any content, scan all 11 files 5 times line by line to confirm accuracy.
  • No content is to be made up or altered without direct permission.
  • Every new idea or scene discussed must be saved in a floating document and made editable.
  • All documents must remain editable and viewable in the project root folder.

Scene File Creation

  • All scene files must be built — no invention or guesswork.
  • Scenes must use the language, emotional tone, and structure as provided in the notes.

XHTML Export Standard

  • All scene exports must be in valid XHTML format.
  • Only ASCII characters may be used (replace ’ with ', –/— with -).
  • Use tabbed indentation throughout the file.
  • All single tags must be self-closing (e.g., <meta />, <br />, <hr />).
  • Structure content using:
    • <h1> for scene titles
    • <h3> for scene headings
    • <p> for dialogue and description
    • <strong> inside <p> for speaker names
  • Include a <style> block for serif font and readable layout.
  • Scenes must render cleanly when served using php artisan serve.

Session Persistence Note

  • Uploaded note files are not persisted between sessions.
  • You must re-upload all 11 Dawnbreakers note files in any new session before continuing work.
  • Once re-uploaded, instruct the assistant to: Scan all 11 files line-by-line 5 times before continuing.
